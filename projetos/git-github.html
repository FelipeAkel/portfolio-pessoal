Projeto Git e GitHub

Ao concluir o curso de Git e GitHub é importante praticar os comandos aprendidos para fixar o conteúdo. Logo, aproveitei para praticar o HTML e CSS para criar um mini portfólio simples como exemplo.

Este projeto teve como objetivo principal executar todos os conhecimentos fixados de Git e o GitHub. Primeiro, Git é o um sistema de controle de versão, baseado em repositório o qual contém todas as versões do código, com critografia e código aberto. Segundo, GitHub é um serviço para gerenciar os repositórios, gratuítos e podemos enviar os projetos para diponibiliza-los para outros DEVs.

Clonando um projeto
  • git clone URL . [Clonando um repositório do GitHub]
  • git remote -v [Verificando repositório está vinculado ao GitHub correto]
  • git branch [Verificando qual branch está realizando as alterações]
  • git checkout -b branch-funcionalidade-x [Criando uma branch nova]
Comandos principais de branch
  • git branch
  • git pull [Baixando os arquivos e modificações do repositório no GitHub]
  • git status [Verificando alterações nos arquivos]
  • git add . [Adicionando arquivos TODOS os arquivos criados e modificados]
  • git commit -a -m "Descrição do Commit" [Realizando o commit das alterações na branch]
  • git push [Enviando as alterações no repositório do GitHub]
