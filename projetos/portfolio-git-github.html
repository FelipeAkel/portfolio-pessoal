Portfólio e Git/GitGub

Ao concluir o curso de Git e GitHub é importante praticar os comandos aprendidos para fixar o conteúdo. Logo, aproveitei para praticar o HTML e CSS para criar um mini portfólio para pra ticar os comandos.

Este projeto teve como objetivo principal executar os principais comandos do Git e fazer o deploy ao GitHub. Primeiro, Git é o um sistema de controle de versão, baseado em repositório o qual contém todas as versões do código, com critografia. Segundo, GitHub é um serviço para gerenciar os repositórios, gratuítos e podemos enviar os projetos para diponibiliza-los para outros DEVs.

Clonando um projeto
  • git clone URL . [Clonando um repositório do GitHub]
  • git remote -v [Verificando repositório está vinculado ao GitHub correto]
  • git branch [Verificando qual branch está realizando as alterações]
  • git checkout -b branch-x [Criando uma branch nova]
Comandos principais de branch
  • git branch
  • git pull [Baixando os arquivos e modificações do repositório no GitHub]
  • git status [Verificando alterações nos arquivos]
  • git add . [Adicionando arquivos TODOS os arquivos criados e modificados]
  • git commit -a -m "Descrição do Commit" [Realizando o commit das alterações na branch]
  • git push [Enviando as alterações no repositório do GitHub]
Comandos principais para gerenciar as versões do projeto
  • git tag [Execute o comando de git responsável por listar todas as tags do projeto]
  • git tag -a v1.0.6 -m "Descrição" [Cria uma nova tag para o repositório]
  • git tag origin v1.0.6 [Execute o comando para enviar SOMENTE a tag nova criada ao GitHub]
  • git tag origin --tags [Execute o comando para enviar TODAS as tags do repositório ao GitHub]
  • git show v1.0.6 [Qual o comando para mostrar as alterações da tag]
  • git checkout v1.0.6 [Qual o comando para mudar as versões do repositório]
